JCL Kernal

Contains JCL2 API kernel. Code that uses the API should access only these classes.

This differs from the JCL1 API in several important ways.

  • The size is much reduced.
  • Code using this distribution will run with just this code.
  • The base implementation is simplified. It should be possible to create JCL implementations that run on early JVMs and in reduced library environments.
  • No bridging implementations are included.
  • Minimal discovery code is included and none is exposed by the API. This allows flexibility in the chose of discovery strategy.

LogFactory now extends LogManager and has been retrofitted with compatibility code. This allows backwards compatibility for components compiled against JCL1.
